# คู่มือติดตั้งระบบตรวจสอบอนุมัติบัตรผ่านยานพาหนะ กองบิน 21

---

## ภาพรวมระบบ

ระบบนี้พัฒนาบน **Google Apps Script (GAS)** โดยใช้ Google Sheets เป็นฐานข้อมูล และแสดงผลผ่าน Web App  
ไม่ต้องติดตั้งโปรแกรมเพิ่มเติม ใช้งานผ่านเบราว์เซอร์ได้ทันที

**ฟังก์ชันหลัก:**
- แสดงรายการ **ดำเนินการแล้ว** (Sheet ที่ 2)
- แสดงรายการ **รออนุมัติ** (Sheet ที่ 3)
- **ค้นหา** ข้อมูลตามเลขทะเบียนรถ (Sheet ที่ 4)

---

## สิ่งที่ต้องเตรียม

| รายการ | รายละเอียด |
|--------|-----------|
| บัญชี Google | บัญชีที่มีสิทธิ์สร้าง Google Sheets และ Apps Script |
| Google Sheets | ไฟล์ข้อมูล (มี 4 Sheet ตามที่กำหนด) |
| เบราว์เซอร์ | Chrome / Edge / Firefox เวอร์ชันล่าสุด |
| อินเทอร์เน็ต | จำเป็น (ระบบโหลด CSS/JS จาก CDN) |

## ขั้นตอนการติดตั้ง

### ขั้นตอนที่ 1 — เตรียม Google Sheets

1. เปิด [Google Sheets](https://sheets.google.com) แล้วสร้างไฟล์ใหม่ หรือใช้ไฟล์ **ตรวจสอบการอนุมัติ.xlsx** ที่มีอยู่แล้ว  
   > วิธีนำเข้าไฟล์ .xlsx: ไปที่ Google Drive → ลากไฟล์วาง → คลิกขวา → เปิดด้วย Google Sheets

2. ตรวจสอบให้มี **Sheet ครบ 4 แผ่น** เรียงตามลำดับดังนี้:

   | ลำดับ Sheet | ชื่อ (ตั้งเองได้) | หน้าที่ |
   |------------|-----------------|--------|
   | Sheet 1 | (สำรอง / ไม่ถูกใช้งาน) | — |
   | Sheet 2 | ดำเนินการแล้ว | ข้อมูลที่อนุมัติแล้ว |
   | Sheet 3 | รออนุมัติ | ข้อมูลที่ยังรออนุมัติ |
   | Sheet 4 | ค้นหาทะเบียน | ข้อมูลสำหรับค้นหาตามทะเบียน |

3. แต่ละ Sheet ควรมีหัวตารางแถวแรก (row 1) และข้อมูลตั้งแต่แถวที่ 2 เป็นต้นไป  
   คอลัมน์ที่ใช้แสดงผล:

   ```
   คอลัมน์ A = ลำดับ
   คอลัมน์ B = ชื่อ สกุล
   คอลัมน์ C = สถานะ
   คอลัมน์ D = เลขทะเบียนรถ
   คอลัมน์ E = รายละเอียดรถ
   คอลัมน์ F = ประเภทรถ
   ```

   > **หมายเหตุ:** ฟังก์ชัน `searchData` ค้นหาจาก **คอลัมน์ B (index 1)** ของ Sheet 4 ด้วยค่าทะเบียนรถ

---

### ขั้นตอนที่ 2 — สร้าง Apps Script Project

1. ใน Google Sheets ที่เตรียมไว้ ไปที่เมนู **ส่วนขยาย → Apps Script**  
   (หรือกด `Extensions → Apps Script`)

2. จะเปิดหน้าต่าง Apps Script Editor ขึ้นมา

3. **ลบโค้ดเดิม** ทั้งหมดในไฟล์ `Code.gs` แล้ว **วางโค้ด** จากไฟล์ `code.gs` ที่ให้มา:

```javascript
function doGet() {
  return HtmlService.createTemplateFromFile('index')
    .evaluate()
    .addMetaTag('viewport', 'width=device-width, initial-scale=1')
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL)
    .setTitle('ระบบตรวจสอบข้อมูลอนุมัติบัตรผ่านยานพาหนะ กองบิน 21')
    .setFaviconUrl('https://cdn-icons-png.flaticon.com/512/1170/1170170.png')
}

function include(filename) {
  return HtmlService.createHtmlOutputFromFile(filename).getContent()
}

function getData() {
  var ss = SpreadsheetApp.getActiveSpreadsheet()
  var sheet = ss.getSheets()[1]
  var range = sheet.getDataRange()
  var values = range.getDisplayValues()
  return values
}

function getData2() {
  var ss = SpreadsheetApp.getActiveSpreadsheet()
  var sheet = ss.getSheets()[2]
  var range = sheet.getDataRange()
  var values = range.getDisplayValues()
  return values
}

function searchData(user) {
  var ss = SpreadsheetApp.getActiveSpreadsheet().getSheets()[3]
  var data = ss.getDataRange().getDisplayValues()
  var result = data.filter(r => r[1] == user.name)
  return result
}
```

### ขั้นตอนที่ 3 — เพิ่มไฟล์ HTML

ต้องสร้างไฟล์ HTML ใน Apps Script Editor **3 ไฟล์** ดังนี้:

#### 3.1 สร้างไฟล์ `index`
1. คลิก **+** ด้านซ้าย → เลือก **HTML**
2. ตั้งชื่อว่า `index`
3. วางเนื้อหาจากไฟล์ `index.html`

#### 3.2 สร้างไฟล์ `css`
1. คลิก **+** → เลือก **HTML**
2. ตั้งชื่อว่า `css`
3. วางเนื้อหาจากไฟล์ `css.html`

#### 3.3 สร้างไฟล์ `DataTable2`
1. คลิก **+** → เลือก **HTML**
2. ตั้งชื่อว่า `DataTable2`
3. วางเนื้อหาจากไฟล์ `DataTable2.html`

> **สำคัญ:** ชื่อไฟล์ต้องตรงกันทุกตัวอักษร (case-sensitive) เพราะ `include()` เรียกชื่อไฟล์โดยตรง

---

### ขั้นตอนที่ 4 — Deploy เป็น Web App

1. ในหน้า Apps Script Editor คลิกปุ่ม **Deploy** (มุมบนขวา) → เลือก **New deployment**

2. คลิกไอคอน ⚙️ ข้าง **Select type** → เลือก **Web app**

3. กำหนดค่า:

   | การตั้งค่า | ค่าที่แนะนำ |
   |-----------|-----------|
   | Description | ตรวจสอบการอนุมัติบัตรผ่าน v1.0 |
   | Execute as | **Me** (บัญชีของคุณ) |
   | Who has access | **Anyone** หรือ **Anyone within [องค์กร]** |

4. คลิก **Deploy**

5. ระบบจะขอ **สิทธิ์ (Permission)** → คลิก **Authorize access** → เลือกบัญชี Google → คลิก **Allow**

6. หลัง Deploy สำเร็จ ระบบจะแสดง **Web app URL** เช่น:
   ```
   https://script.google.com/macros/s/AKfycb.../exec
   ```
   > **คัดลอก URL นี้เก็บไว้** สำหรับแจกจ่ายให้ผู้ใช้งาน

---

### ขั้นตอนที่ 5 — ทดสอบระบบ

1. เปิด Web app URL ที่ได้จากขั้นตอนที่ 4 ในเบราว์เซอร์
2. ตรวจสอบว่าหน้าเว็บโหลดขึ้นมาพร้อมข้อมูล
3. ทดสอบแต่ละฟังก์ชัน:

   - ✅ หน้าแรกแสดงข้อมูล **ดำเนินการแล้ว** จาก Sheet 2
   - ✅ คลิก **รออนุมัติ** แสดงข้อมูลจาก Sheet 3  
   - ✅ พิมพ์เลขทะเบียนรถในช่องค้นหา แล้วกด **Search** แสดงผลลัพธ์จาก Sheet 4

---

## การอัปเดตโค้ดหลัง Deploy แล้ว

เมื่อแก้ไขโค้ด ต้อง Deploy ใหม่ทุกครั้ง:

1. คลิก **Deploy** → **Manage deployments**
2. คลิกไอคอน ✏️ ที่ deployment ที่มีอยู่
3. เปลี่ยน **Version** เป็น **New version**
4. คลิก **Deploy**

> URL เดิมยังคงใช้ได้ ไม่จำเป็นต้องแจก URL ใหม่

## การแก้ไขปัญหาเบื้องต้น

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ |
|-------|------------------|--------|
| ไม่แสดงข้อมูล | Sheet ไม่ครบ 4 แผ่น | ตรวจสอบจำนวน Sheet |
| ข้อมูลผิด Sheet | ลำดับ Sheet ไม่ถูกต้อง | เรียงลำดับ Sheet ใหม่ |
| ค้นหาไม่เจอข้อมูล | ทะเบียนไม่ตรงกับคอลัมน์ B ใน Sheet 4 | ตรวจสอบข้อมูลใน Sheet 4 |
| หน้าเว็บไม่โหลด | ไม่มีอินเทอร์เน็ต / CDN ล่ม | ตรวจสอบการเชื่อมต่อ |
| Error: Authorization | ยังไม่ได้ให้ Permission | ทำขั้นตอน Deploy ใหม่แล้วกด Allow |
| ไฟล์ HTML หาไม่เจอ | ชื่อไฟล์ผิด | ตรวจสอบชื่อไฟล์ให้ตรงตามที่กำหนด |
