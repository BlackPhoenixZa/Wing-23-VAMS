# คู่มือการติดตั้ง ระบบฐานทะเบียนรถ WING 23
**DATABASE VEHICLE PERMIT WING 23**

---

## ภาพรวมระบบ

ระบบนี้เป็น Web App สร้างด้วย **Google Apps Script (GAS)** เชื่อมต่อกับ **Google Sheets** เพื่อแสดงข้อมูลทะเบียนรถในรูปแบบตารางค้นหาได้ (DataTable) ผ่านเว็บเบราว์เซอร์

**Technology Stack:**
- Google Apps Script (Backend + Hosting)
- Google Sheets (ฐานข้อมูล)
- DataTables 1.11.4 (ตารางแสดงข้อมูล)
- Bootstrap 5.1.3 (UI Framework)
- jQuery 3.5.1
- Font Awesome 6.0.0
- SweetAlert2 (Popup)

---

## ขั้นตอนการติดตั้ง

### ขั้นตอนที่ 1 — เตรียม Google Sheets (ฐานข้อมูล)

1. เปิด [Google Sheets](https://sheets.google.com) แล้วสร้าง Spreadsheet ใหม่
2. เปลี่ยนชื่อ Sheet เป็น **`Data`** (ตัวพิมพ์ใหญ่ D)
3. กำหนดหัวคอลัมน์แถวที่ 1 ตามลำดับนี้:

| คอลัมน์ | หัวคอลัมน์ |
|--------|-----------|
| A | ประเภทรถ |
| B | เลขทะเบียนรถ |
| C | ชื่อ สกุล |
| D | เบอร์โทรติดต่อ |
| E | ชื่อ สกุล (บุคคลติดต่อฉุกเฉิน) |
| F | เบอร์โทรบุคคลติดต่อฉุกเฉิน |
| G | เลขประจำตัวประชาชน |
| H | ประเภทรถ |
| I | ลักษณะรถ |
| J | จังหวัด |
| K | ยี่ห้อรถ |
| L | แบบรถ |
| M | สีรถ |
| N | วันเสียภาษีรถ |
| O | วันครบกำหนดเสียภาษีรถ |
| P | ที่อยู่ |

4. คัดลอก **Spreadsheet ID** จาก URL ของ Sheets:
   ```
   https://docs.google.com/spreadsheets/d/[SPREADSHEET_ID]/edit
   ```

---

### ขั้นตอนที่ 2 — สร้าง Google Apps Script Project

1. ไปที่ [script.google.com](https://script.google.com) แล้วคลิก **"New project"**
2. ตั้งชื่อโปรเจกต์ เช่น `DATABASE VEHICLE PERMIT WING 23`

---

### ขั้นตอนที่ 3 — นำเข้าไฟล์โค้ด

#### 3.1 ไฟล์ `Code.gs`
คลิกที่ไฟล์ `Code.gs` (ไฟล์เริ่มต้น) แล้วแทนที่เนื้อหาทั้งหมดด้วยโค้ดจากไฟล์ `code.gs` ในโปรเจกต์

**สำคัญ:** แก้ไข Spreadsheet ID บรรทัดนี้ให้ตรงกับ Sheets ที่สร้างในขั้นตอนที่ 1:
```javascript
var id = 'SPREADSHEET_ID_ของคุณ'
```

#### 3.2 ไฟล์ HTML
ต้องสร้างไฟล์ HTML แยกจำนวน 3 ไฟล์ โดยคลิก **"+"** ด้านซ้าย → **"HTML"**:

| ชื่อไฟล์ | เนื้อหา |
|---------|---------|
| `index` | โครงสร้างหน้าเว็บหลัก (จากไฟล์ `index.html`) |
| `css` | CSS และ Library styles (จากไฟล์ `css.html`) |
| `js` | JavaScript และ DataTable config (จากไฟล์ `js.html`) |

> ⚠️ ชื่อไฟล์ต้องตรงตามนี้ทุกตัวอักษร ระบบใช้ `include('css')` และ `include('js')` เรียกไฟล์

---

### ขั้นตอนที่ 4 — Deploy เป็น Web App

1. คลิก **"Deploy"** (มุมขวาบน) → **"New deployment"**
2. คลิกไอคอน ⚙️ → เลือก **"Web app"**
3. ตั้งค่าดังนี้:

| การตั้งค่า | ค่าที่เลือก |
|-----------|------------|
| Description | `v1.0` หรือตามต้องการ |
| Execute as | `Me (อีเมลของคุณ)` |
| Who has access | `Anyone` (ถ้าต้องการให้ทุกคนเข้าได้) หรือ `Anyone within [องค์กร]` |

4. คลิก **"Deploy"**
5. ระบบจะขอสิทธิ์ — คลิก **"Authorize access"** แล้วอนุญาตการเข้าถึง Google Sheets
6. คัดลอก **Web app URL** ที่ได้รับ — นี่คือ URL ของระบบ

---

### ขั้นตอนที่ 5 — ทดสอบระบบ

1. เปิด URL จากขั้นตอนที่ 4 ในเบราว์เซอร์
2. ระบบจะแสดง Spinner (วงหมุน) ขณะโหลดข้อมูล
3. เมื่อโหลดเสร็จ ตารางข้อมูลจะแสดงพร้อมฟังก์ชัน:
   - **ค้นหา** — กรองข้อมูลแบบ real-time
   - **Export Excel** — ดาวน์โหลดข้อมูลเป็น .xlsx
   - **คัดลอก / พิมพ์** — ผ่านปุ่ม Dropdown
   - **แสดงแถว** — เลือกจำนวนแถวที่แสดง (5, 10, 25, 50, 100, ทั้งหมด)
   - **Responsive** — รองรับหน้าจอมือถือ

---

## การอัปเดตโค้ดหลัง Deploy แล้ว

เมื่อแก้ไขโค้ด ต้อง Deploy เวอร์ชันใหม่ทุกครั้ง:

1. คลิก **"Deploy"** → **"Manage deployments"**
2. คลิกไอคอน ✏️ ที่ deployment ที่ต้องการแก้ไข
3. เปลี่ยน Version เป็น **"New version"**
4. คลิก **"Deploy"**

> ⚠️ ถ้าเลือก "New version" URL จะยังเดิม แต่ถ้าสร้าง deployment ใหม่จะได้ URL ใหม่

## การแก้ไขข้อมูล

ข้อมูลทั้งหมดจัดการผ่าน **Google Sheets** โดยตรง:
- เพิ่ม/แก้ไข/ลบข้อมูลใน Sheet `Data`
- การเปลี่ยนแปลงจะสะท้อนทันทีเมื่อผู้ใช้โหลดหน้าเว็บใหม่

## โครงสร้างไฟล์

```
Google Apps Script Project/
├── Code.gs          ← Backend: doGet(), getData(), include()
├── index.html       ← โครงสร้าง HTML หลัก
├── css.html         ← CSS Libraries และ Custom styles
└── js.html          ← JavaScript, DataTable configuration
```

---

## การแก้ไขปัญหาเบื้องต้น

| ปัญหา | วิธีแก้ไข |
|------|----------|
| ตารางไม่แสดงข้อมูล | ตรวจสอบ Spreadsheet ID ใน `code.gs` และชื่อ Sheet ต้องเป็น `Data` |
| หน้าเว็บแสดง Error permission | Re-authorize โดยไปที่ Deploy → Manage deployments แล้ว deploy ใหม่ |
| แก้ไขโค้ดแล้วไม่เปลี่ยน | ต้อง Deploy เวอร์ชันใหม่ทุกครั้ง |
| Spinner หมุนไม่หยุด | เปิด DevTools (F12) ดู Console errors ตรวจสอบ Sheets permission |
